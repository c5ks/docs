## Overview

Async executions are used for models that take longer than 30 seconds — typically image generation, video generation, and audio synthesis. You submit the request, get an execution ID immediately, and receive the result via webhook or by polling.

## Submit an async execution

```bash
curl -X POST https://api.modelroute.ai/v1/executions \
  -H "Authorization: Bearer sk_your_api_key_here" \
  -H "Content-Type: application/json" \
  -H "Idempotency-Key: async-video-001" \
  -d '{
    "model": "minimax-video-01",
    "input": {
      "prompt": "A drone shot flying over a mountain range at golden hour",
      "duration": 5
    }
  }'
```

### Immediate response (202 Accepted)

The API returns immediately with the execution in `PENDING` state:

```json
{
  "id": "exec_b2c3d4e5-f678-9012-abcd-ef1234567890",
  "status": "PENDING",
  "model": "minimax-video-01",
  "mode": "ASYNC",
  "idempotency_key": "async-video-001",
  "estimated_cost": "0.250",
  "created_at": "2026-03-20T10:30:00Z"
}
```

## Option 1: Webhooks (recommended)

Register a webhook endpoint to receive completion events automatically. See [Webhook Setup](/webhooks/setup) for configuration.

When the execution completes, ModelRoute sends a `POST` to your webhook URL:

```json
{
  "id": "evt_c3d4e5f6-7890-1234-abcd-ef1234567890",
  "event_type": "execution.completed",
  "timestamp": "2026-03-20T10:32:15Z",
  "version": "1.0",
  "data": {
    "id": "exec_b2c3d4e5-f678-9012-abcd-ef1234567890",
    "status": "COMPLETED",
    "model": "minimax-video-01",
    "cost": "0.220",
    "latency_ms": 135000,
    "result": {
      "file_id": "file_d4e5f678-9012-3456-abcd-ef1234567890",
      "content_type": "video/mp4",
      "duration": 5.0
    },
    "created_at": "2026-03-20T10:30:00Z",
    "completed_at": "2026-03-20T10:32:15Z"
  }
}
```

If the execution fails, you receive an `execution.failed` event instead:

```json
{
  "id": "evt_d4e5f678-9012-3456-abcd-ef1234567890",
  "event_type": "execution.failed",
  "timestamp": "2026-03-20T10:31:00Z",
  "version": "1.0",
  "data": {
    "id": "exec_b2c3d4e5-f678-9012-abcd-ef1234567890",
    "status": "FAILED",
    "model": "minimax-video-01",
    "error_code": "GENERATION_FAILED",
    "error_message": "Generation failed. Please retry"
  }
}
```

## Option 2: Polling

Poll the execution endpoint until you reach a terminal state (`COMPLETED`, `FAILED`, `CANCELLED`, `EXPIRED`):

<CodeGroup>
```bash curl
# Poll every 5 seconds
while true; do
  RESPONSE=$(curl -s -X GET \
    https://api.modelroute.ai/v1/executions/exec_b2c3d4e5-f678-9012-abcd-ef1234567890 \
    -H "Authorization: Bearer sk_your_api_key_here")

  STATUS=$(echo "$RESPONSE" | jq -r '.status')
  echo "Status: $STATUS"

  if [ "$STATUS" = "COMPLETED" ] || [ "$STATUS" = "FAILED" ] || \
     [ "$STATUS" = "CANCELLED" ] || [ "$STATUS" = "EXPIRED" ]; then
    echo "$RESPONSE" | jq .
    break
  fi

  sleep 5
done
```

```python Python
import requests
import time

execution_id = "exec_b2c3d4e5-f678-9012-abcd-ef1234567890"
headers = {"Authorization": "Bearer sk_your_api_key_here"}

while True:
    response = requests.get(
        f"https://api.modelroute.ai/v1/executions/{execution_id}",
        headers=headers
    )
    data = response.json()
    print(f"Status: {data['status']}")

    if data["status"] in ("COMPLETED", "FAILED", "CANCELLED", "EXPIRED"):
        print(data)
        break

    time.sleep(5)
```

```typescript TypeScript
const executionId = "exec_b2c3d4e5-f678-9012-abcd-ef1234567890";

async function pollExecution() {
  while (true) {
    const response = await fetch(
      `https://api.modelroute.ai/v1/executions/${executionId}`,
      { headers: { "Authorization": "Bearer sk_your_api_key_here" } }
    );
    const data = await response.json();
    console.log(`Status: ${data.status}`);

    if (["COMPLETED", "FAILED", "CANCELLED", "EXPIRED"].includes(data.status)) {
      return data;
    }

    await new Promise(resolve => setTimeout(resolve, 5000));
  }
}
```
</CodeGroup>

<Warning>
  Polling is simpler but less efficient than webhooks. Use webhooks in production to avoid unnecessary API calls and reduce latency between completion and notification.
</Warning>

## Cancelling an async execution

Cancel a pending or processing execution to release the balance hold:

```bash
curl -X POST https://api.modelroute.ai/v1/executions/exec_b2c3d4e5-f678-9012-abcd-ef1234567890/cancel \
  -H "Authorization: Bearer sk_your_api_key_here"
```

```json
{
  "id": "exec_b2c3d4e5-f678-9012-abcd-ef1234567890",
  "status": "CANCELLED",
  "cost": "0.000"
}
```

<Note>
  Cancellation is best-effort. If the provider has already completed the execution, the result may still be delivered and billed.
</Note>
