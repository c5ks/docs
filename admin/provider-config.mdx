## Anatomy of a provider config

Every provider is defined in `configs/providers/<slug>/config.yaml`. Here's a complete example:

```yaml
provider:
  slug: kie                        # Unique identifier (lowercase, hyphens OK)
  name: Kie AI                     # Display name
  description: Competitive AI API  # Optional description
  base_url: https://api.kie.ai     # Base URL for all API calls

auth:
  type: bearer                     # bearer | api_key | basic | pooled | none
  secret_name: kie_api_key         # Secret name in your secrets manager

defaults:
  timeout_seconds: 180             # Default timeout for all models
  max_retries: 3                   # Retry count on transient failures
  retry_delay_ms: 2000             # Base delay between retries

# Optional: provider-level error classification
error_mapping:
  status_codes:
    400: INPUT_VALIDATION_ERROR
    401: MODEL_UNAVAILABLE
    403: CONTENT_MODERATION_REJECTED
    429: RATE_LIMITED
    503: MODEL_OVERLOADED
  body_patterns:
    - match: "content_policy_violation"
      code: CONTENT_MODERATION_REJECTED
    - match: "rate_limit"
      code: RATE_LIMITED
  default: GENERATION_FAILED

models:
  - slug: google/nano-banana
    name: Nano Banana Pro
    category: image-generation      # image-generation | video-generation | text-generation | audio-generation
    execution_mode: ASYNC           # ASYNC (always — we don't expose sync)
    timeout_seconds: 180            # Override default

    # How to submit the job
    submit:
      method: POST
      path: /api/v1/jobs/createTask
      content_type: application/json
      job_id_path: "$.data.taskId"  # JSONPath to extract job ID from response

    # How to poll for results
    poll:
      method: GET
      path: /api/v1/jobs/recordInfo?taskId={{ .job_id }}
      interval_seconds: 2
      max_attempts: 90
      initial_delay_seconds: 3
      success_condition: 'state == "success"'
      failure_condition: 'state == "fail"'
      output_path: "$.data"

    # Request body template (Go template syntax)
    body_template: |
      {
        "model": "google/nano-banana",
        "input": {
          "prompt": "{{ escape .input.prompt }}",
          "output_format": "{{ default .input.output_format "png" }}"
        }
      }

    # Input validation (JSON Schema)
    input_schema:
      type: object
      required: [prompt]
      properties:
        prompt:
          type: string
          maxLength: 5000
        output_format:
          type: string
          enum: [png, jpeg]
          default: png

    # Response parsing
    response:
      success_condition: "code == 200"
      error_mapping:
        message: "$.msg"
        code: "$.code"
      output_mapping:
        task_id: "$.data.taskId"
        images: "$.data.resultJson"
      usage_mapping:
        images: 1

    # Pricing
    pricing:
      provider_cost:
        per_image: 0.04
      customer_price:
        per_image: 0.06
      platform_fee:
        percent: 5

    # File handling mode
    file_handling: presigned_url    # presigned_url | inline_base64 | raw_passthrough
```

## Authentication types

| Type | Config | How it works |
|-|-|-|
| `bearer` | `secret_name` | Sends `Authorization: Bearer <secret>` |
| `api_key` | `secret_name`, `header` | Sends secret in custom header (default `X-API-Key`) |
| `basic` | `secret_name` | Sends `Authorization: Basic <secret>` |
| `pooled` | `pooled.pool_strategy`, `pooled.account_source` | Rotates through multiple accounts from database |
| `none` | — | No authentication |

## Body templates

Body templates use Go's `text/template` syntax with extra helpers:

| Function | Usage | Description |
|-|-|-|
| `escape` | `{{ escape .input.prompt }}` | JSON-escapes a string |
| `default` | `{{ default .input.format "png" }}` | Falls back to default value |
| `toJSON` | `{{ toJSON .input }}` | Marshals value to JSON |

Available context variables:
- `.input` — the customer's input parameters
- `.execution_id` — execution UUID
- `.request_id` — tracing request ID
- `.model` — model slug
- `.provider` — provider slug

## Template inheritance

Providers with similar APIs can inherit from templates:

```yaml
# configs/provider_templates/openai-compatible.yaml
provider:
  slug: openai-compatible
auth:
  type: bearer
defaults:
  timeout_seconds: 120
models:
  - request:
      method: POST
      content_type: application/json
```

```yaml
# configs/providers/my-provider/config.yaml
extends: openai-compatible         # Inherit auth, defaults, request config

provider:
  slug: my-provider
  name: My Provider
  base_url: https://api.myprovider.com

auth:
  secret_name: my_provider_key     # Override just the secret

models:
  - slug: my-model
    name: My Model
    category: image-generation
    execution_mode: ASYNC
    # request config inherited from template
```

## Passthrough mode

For providers like 11Labs that need the exact customer payload forwarded:

```yaml
passthrough: true    # Skip body template, input validation, response normalization
```

## Validation

Always validate before deploying:

```bash
make validate-configs
```

This checks:
- Required fields present
- Referenced secrets exist
- Body templates parse correctly
- Poll conditions compile
- Cross-references between models, mappings, and routes are valid
